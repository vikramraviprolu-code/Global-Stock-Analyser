Privacy & Compliance

Last updated: 2026-05-01 · v0.20.0

TL;DR

  • ✅ No cookies. Zero. Not even functional.
  • ✅ No server-side storage of personal data. Server only caches market data (prices, fundamentals, FX) — never tied to you.
  • ✅ No analytics, no third-party trackers, no telemetry.
  • ✅ No login / no account / no PII collected.
  • ✅ No AI / LLM / ML models used anywhere. All "AI-like" features (sentiment, recommendation, scoring) are transparent rule-based heuristics.
  • ✅ Watchlists, portfolio, alerts, prefs live in your browser's localStorage only — on this device, never transmitted, never sync'd.
  • ✅ You are in full control. Reset everything with one click on the Settings page.

GDPR / UK GDPR compliance

Lawful basis (Article 6)

EquityScope does not process any personal data on the server side. Browser-local storage of your preferences relies on the lawful basis of consent (Article 6(1)(a)) where you explicitly accept the consent banner, or legitimate interest for strictly-necessary settings (e.g. remembering "Compact mode") under PECR / ePrivacy "essential" exemption.

Data subject rights (Articles 15–22)

Because no personal data is held server-side, the following rights are satisfied through self-service controls in your browser:

  • Article 15 — Right to access: Open DevTools → Application → Local Storage to view every stored value verbatim.
  • Article 16 — Right to rectification: Edit any stored value via the corresponding page (Settings, Watchlists, Portfolio, etc.).
  • Article 17 — Right to erasure: Settings → "Reset all preferences" wipes every equityscope.* key. Or DevTools → Clear local storage.
  • Article 20 — Right to data portability: Portfolio + Holdings + Source-Audit pages all expose CSV + JSON export. Watchlist export is on the roadmap (v0.21.0).
  • Article 21 — Right to object: Decline the consent banner; localStorage features are then disabled.
  • Article 22 — No automated decision-making: EquityScope produces transparent, deterministic, rule-based recommendations. No ML model is involved in any output that affects you. (See AI Act section below.)

Data Protection Officer (DPO)

This project does not require a DPO under Article 37 (no large-scale or systematic processing). Coordinated disclosure / privacy queries: open an issue at the project repository (link in the footer).

International transfers

Market data fetched from Yahoo Finance + Stooq passes between your local machine and those public free endpoints. The app never transmits any of your stored watchlists / portfolio / alerts to any third party. Endpoints contacted on your behalf:

  • finance.yahoo.com — prices, fundamentals, news, calendar (USA)
  • stooq.com — OHLCV CSV (Poland; gated for many tickers)
  • github.com/FiloSottile/mkcert — only during initial install of the local TLS CA (one-off)

Children

EquityScope is not directed at children under 16. No PII is collected, so no parental consent flow is required (Article 8 not triggered).

EU AI Act compliance (Regulation 2024/1689)

Does EquityScope use an AI system?

No. EquityScope contains no machine-learning model, no large language model, no neural network, and no remote AI API call.

What about features called "AI-like" or "smart"?

Every label, score, sentiment classification, recommendation, and topic clustering in the app is the output of transparent rule-based logic with explicit weights you can read in the source code. Specifically:

  • Headline sentiment (News page) — keyword-counting against fixed positive / negative dictionaries. Not an LLM, not a fine-tuned model. See providers/news.py for the exact word lists.
  • Topic clustering (News page) — first-match dictionary lookup against fixed keyword sets. Not a clustering algorithm.
  • Value / Momentum / Quality / Risk / Data Confidence scores — additive integer rules with explicit point weights documented in calc/scoring.py + the README + the in-app explainer drawers.
  • Scenario recommendation (Buy / Watch / Avoid) — deterministic if/else logic over scores. Same inputs → same output. No probabilistic model.

Why this matters under the AI Act

  • Article 3 — definition: "AI system" requires inference and adaptiveness based on objectives. Rule-based scoring with hand-crafted weights is excluded by Recital 12.
  • Article 5 — prohibited practices: No applicability — we don't do social scoring, biometrics, manipulation, or law-enforcement profiling.
  • Article 6 / Annex III — high-risk systems: No applicability — no credit-scoring, no employment, no critical infrastructure.
  • Article 50 — transparency obligations: Although not required since no AI system is used, we voluntarily disclose every heuristic on the explainer drawers ("?" buttons next to every metric / score) to maintain auditability.
  • Article 22 GDPR — automated decision-making: The "Buy / Watch / Avoid" recommendation is informational only. It does not produce legal effects on you, and you remain free to ignore or override it. This is not Article-22 ADM.

Will this change in future versions?

If a future version ever introduces an actual AI system (e.g. an LLM-summarised news digest), this notice will be revised before shipping. Current PRD constraint forbids paid AI APIs; any local LLM addition would arrive with full Article 50 transparency labelling, output flagging, and an opt-out.

Security

For security details + threat model see SECURITY.md. Highlights:

  • HTTPS via mkcert local Certificate Authority (no self-signed warnings)
  • Bind to 127.0.0.1 by default — no LAN exposure
  • Strict Host-header allow-list rejects unknown hostnames
  • CSRF protection on every state-changing endpoint (/api/shutdown, /api/settings/clear-cache) via Origin / Referer + loopback check
  • X-Frame-Options DENY, strict Content-Security-Policy, X-Content-Type-Options nosniff, Referrer-Policy no-referrer, restrictive Permissions-Policy
  • Multi-source price verification (Stooq + yfinance cross-validation) earns a green ✓ badge when both agree
  • Server header replaced with EquityScope — no Werkzeug version banner
  • Coordinated vulnerability disclosure via /.well-known/security.txt (RFC 9116)
  • Self-hosted vendor JS (Lightweight Charts) with Subresource Integrity hashing
